# Promise Execution

## Promise Object Internals

When `new Promise(executor)` is created, V8 allocates a promise object with:
- **state**: `pending` | `fulfilled` | `rejected`
- **result**: the resolved value or rejection reason
- **fulfill reactions**: list of `.then` handlers waiting on success
- **reject reactions**: list of `.catch` / `.then(_, onRejected)` handlers
- **handled flag**: whether a rejection handler exists (unhandled rejections trigger warnings)

## How `.then` Works

- Calling `.then(handler)` creates a **PromiseReaction record** and adds it to the promise's reaction list
- When the promise resolves, its fulfill reactions are scheduled — their handlers are pushed to the **microtask queue**, not called immediately
- This is why `.then` callbacks are always async, even if the promise is already resolved

## Execution Order

1. Synchronous code runs to completion (empties the call stack)
2. **Microtask queue** is drained entirely — all `.then` / `queueMicrotask` callbacks run
3. One macrotask (e.g. `setTimeout` callback) is picked up
4. Repeat: drain microtasks again before the next macrotask

- Microtasks always run before the next macrotask — chaining many `.then`s all flush before any `setTimeout` fires
- The call stack must be empty before the event loop checks either queue

## Quick Example

```js
console.log("1");
Promise.resolve().then(() => console.log("2"));
setTimeout(() => console.log("3"), 0);
console.log("4");
// Output: 1, 4, 2, 3
```

`1` and `4` are synchronous. `2` is a microtask, runs before the `setTimeout` macrotask `3`.
