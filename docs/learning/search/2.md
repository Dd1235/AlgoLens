# Evaluation Metrics for Search & Recommender Systems

## Online vs Offline Metrics

- **Offline**: evaluated against a labeled dataset, no live users — faster to iterate
- **Online**: measured on real user behavior (CTR, dwell time) — A/B testing is the standard approach
- Offline metrics should correlate with online metrics, but the gap can be large in practice

## Precision & Recall @K (Order-Unaware)

- Evaluated on the top-K results returned
- **Precision@K**: of the K results shown, what fraction are relevant? — measures result quality
- **Recall@K**: of all relevant documents, what fraction appear in the top K? — measures coverage
- **F1@K**: harmonic mean of precision and recall

- Limitation: **order-unaware** — ranking a relevant doc at position 1 vs position K scores the same

## MRR — Mean Reciprocal Rank (Order-Aware)

- Measures how high the **first** relevant result appears
- For each query, score = `1 / rank` of the first relevant result (e.g. rank 1 → 1.0, rank 3 → 0.33)
- Averaged across all queries: `MRR = (1/|Q|) × Σ (1 / rank_q)`
- Good for tasks where finding one correct answer matters (e.g. QA, navigational queries)
- Limitation: ignores all relevant results after the first

## MAP — Mean Average Precision (Order-Aware)

- Averages precision at each rank position where a relevant doc appears
- Rewards systems that rank all relevant docs high, not just the first one
- `AP` for one query = average of Precision@k for each k where result k is relevant
- `MAP` = mean of AP across all queries
- More comprehensive than MRR when there are multiple relevant documents per query

## NDCG — Normalized Discounted Cumulative Gain (Order-Aware, Graded)

- Handles **graded relevance** (e.g. 0 = irrelevant, 1 = somewhat relevant, 2 = very relevant) — unlike the binary relevant/not of MRR and MAP
- **DCG@K**: sums relevance scores with a log discount for lower positions — top results matter more
- **NDCG@K**: DCG normalized by ideal DCG (best possible ranking), giving a score in [0, 1]
- Standard metric for modern search and recommendation benchmarks

## Quick Comparison

| Metric | Order-Aware | Graded Relevance | Best For |
|--------|-------------|-----------------|----------|
| Precision/Recall@K | No | No | Quick baseline |
| MRR | Yes | No | Single correct answer |
| MAP | Yes | No | Multiple binary-relevant docs |
| NDCG@K | Yes | Yes | Full ranking quality |
