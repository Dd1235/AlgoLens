# Vector-Based Similarity Search

## Sparse vs Dense Vectors

- **TF-IDF / BM25**: sparse vector methods — each dimension is a vocabulary term, most values are zero
- **SBERT (Sentence-BERT)**: dense vector method — embeds meaning into a fixed-size vector (~768 dims, all non-zero)
- Sparse = exact keyword matching; dense = semantic similarity ("car" ≈ "automobile")

## TF-IDF Recap

- `TF(t, D) = count(t in D) / total terms in D` — how often the term appears, normalized by doc length
- `IDF(t) = log(N / df(t))` — penalizes terms that appear in many documents (e.g. "is" → IDF ≈ 0)
- `TF-IDF(t, D) = TF × IDF` — high score means the term is frequent in this doc but rare globally

```python
import numpy as np

docs = [doc_a, doc_b, doc_c]

def tfidf(word, sentence):
    tf = sentence.count(word) / len(sentence)
    idf = np.log10(len(docs) / sum(1 for doc in docs if word in doc))
    return round(tf * idf, 4)
```

## BM25 (Best Match 25)

- BM25 is the modern standard — a **probabilistic** retrieval model, not just a heuristic like TF-IDF
- Key improvements over TF-IDF:
  - **Term saturation**: extra repetitions of a term give diminishing returns (spammy docs penalized)
  - **Document length normalization**: long docs don't get unfair boosts just from having more words

**Tuning parameters:**
- `k` (~1.25): controls how fast TF saturates — higher k = more weight to raw frequency
- `b` (~0.75): controls length normalization strength — `b=1` fully normalizes, `b=0` ignores length

**Formula (simplified):**

$$
BM25(D, q) = \sum_{t \in q} IDF(t) \cdot \frac{TF(t,D) \cdot (k+1)}{TF(t,D) + k \cdot (1 - b + b \cdot \frac{|D|}{d_{avg}})}
$$

Where `|D|` is the current doc length and `d_avg` is the average doc length across the corpus.

## When to Use Which

- **BM25**: default choice for keyword search — fast, interpretable, strong baseline
- **TF-IDF**: simpler, still useful for quick prototypes or ML feature pipelines
- **Dense (SBERT)**: needed when semantic understanding matters — "find papers about climate change" should match "global warming"
- Hybrid: many production systems combine BM25 (recall) with dense re-ranking (precision)
