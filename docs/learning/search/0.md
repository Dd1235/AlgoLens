# TF-IDF

## What It Is

- TF-IDF (**Term Frequency – Inverse Document Frequency**) is a classic scoring algorithm for ranking documents by relevance to a query
- Rewards terms that appear often in a document but rarely across the whole corpus — "common in this doc, rare overall" = high signal

## Term Frequency (TF)

- How often a query token appears in a given document
- More occurrences → higher TF score for that document
- Usually normalized by document length so longer docs aren't unfairly boosted

## Inverse Document Frequency (IDF)

- Measures how rare a term is across all documents
- A word like "the" appears in every document → low IDF, contributes little
- A word like "photosynthesis" appears in few documents → high IDF, highly discriminative
- Formula: `IDF(t) = log(N / df(t))` where N = total docs, df(t) = docs containing term t

## TF-IDF Score

- `TF-IDF(t, d) = TF(t, d) × IDF(t)`
- Final document score = sum of TF-IDF scores across all query terms
- Higher score → more relevant document

## Quick Example

Query: "graph algorithm"

- Doc A: mentions "graph" 5 times, "algorithm" 3 times — both rare across corpus → high score
- Doc B: mentions "algorithm" once, but "algorithm" appears in 90% of docs → low IDF → low contribution

## Limitations

- No semantic understanding — "car" and "automobile" are treated as unrelated
- Doesn't capture word order or context
- BM25 is the modern improvement: adds document length normalization and term saturation (extra repetitions give diminishing returns)

## Where It's Used

- Classic search engines, early Google-era IR systems
- Still used as a baseline in information retrieval and as a feature in ML ranking pipelines
